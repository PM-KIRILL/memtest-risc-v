# MemTest86+ с поддержкой risc-v архитектуры

Это глубоко переработанная версия [MemTest86+](http://www.memtest.org/) v5.1, адаптированная для работы на различных аппаратных архитектурах.

> [!CAUTION]
> **ОТКАЗ ОТ ОТВЕТСТВЕННОСТИ:** Используйте это программное обеспечение на свой страх и риск. Не запускайте его на реальном оборудовании, если вы не понимаете принципы его работы и возможные последствия.

> [!WARNING]
> **СТАТУС ПРОЕКТА:** Продукт находится в стадии активной разработки (Work-in-Progress). Возможны ложноположительные или ложноотрицательные результаты тестов.

## 📋 Описание порта RISC-V

Данный репозиторий содержит порт для архитектуры **RISC-V**, протестированный на сборке RocketChip для FPGA-платы с AliExpress. Поскольку плата оснащена стандартным слотом DDR2-SODIMM, тестирование памяти здесь является практической необходимостью, а не просто демонстрацией возможностей.

**Особенности реализации:**
*   **Динамический парсинг:** Я постарался реализовать парсинг FDT (*Flattened Device Tree*) вместо жесткой привязки адресов памяти (hardcoding).
*   **Ограничения:** В коде всё еще могут встречаться «магические числа». 
*   **Важно:** Текущая версия **не обрабатывает зарезервированные области памяти**. В моем софт-процессоре их нет, но на серийных CPU это может привести к повреждению данных или сбоям системы.

## 🚀 Использование

### Сборка
Для компиляции выполните:
```bash
make ARCH=riscv
```
*Примечание: Если инструменты сборки не найдены, отредактируйте пути в файле `arch/riscv/config.mk`.*

### Результаты сборки
В директории появятся следующие файлы:
*   `memtest_shared` — объект ELF с отладочной информацией для GDB.
*   `memtest_shared.bin` — чистый бинарный образ (raw binary) для загрузки в RAM.
*   `memtest.uboot` — образ для U-Boot, имитирующий загрузку ядра Linux.

### Запуск через U-Boot
Пример команды для загрузки на моей плате:
```bash
run mmcsetup; run fdtsetup; fdt set /chosen bootargs "console=ttyS0"; fatload mmc 0:1 82000000 memtest.uboot; bootm fdt; bootm 82000000 - ${fdtaddr}
```

### Ручной запуск
Поместите `memtest_shared.bin` по любому адресу в оперативной памяти. Настройте регистры:
*   `$pc` — адрес загрузки образа.
*   `$a0` — **hart id** (на данный момент не используется, так как SMP не поддерживается, оставлено для совместимости).
*   `$a1` — адрес загрузки **FDT** в памяти.
*   *Примечание:* Параметры командной строки (например, `console=ttyS0`) считываются из узла `/chosen/bootargs` в FDT.
